## 当前阶段一句话结论

系统已经是**形式稳定的 MVP**，但还不是**事实稳健的 MVP**。

---

## 四大核心问题（速览）

### 1. candidate_profile 抽取质量不够
- **表现**：skills / projects / experiences 混入大量原始整句，标题和正文没有分干净，missing_items 识别不准
- **本质**：更像"切文本"，不是真正结构化
- **优先级**：最高（是后续所有模块的基础）

### 2. JD 分析整体不错，但还不够可执行
- **表现**：能总结岗位要求，但关键词、能力、工程要求混在一起，没有优先级
- **本质**：更像"总结"，还不是"可直接驱动下游的拆解"
- **优先级**：中

### 3. match 分析已经有价值，但仍偏关键词驱动
- **表现**：能结合项目做匹配分析，但容易从弱证据推到强结论，证据等级没有区分
- **本质**：词匹配偏多，能力匹配偏少
- **优先级**：中（依赖 candidate_profile 改善后再优化）

### 4. rewrite 是当前最高风险模块
- **表现**：生成了原文没有的量化数字（如"降低串库风险约30%"），引入了原文未提及的功能描述
- **本质**：输出表达变强了，但事实边界没有守死
- **优先级**：高（直接影响候选人使用安全性）

---

## 优化优先级

1. **先优化 candidate_profile**：把中间结构化做干净
2. **再收紧 rewrite**：优先守住事实边界，禁止生成未经证实的数字和功能
3. **再升级 match**：从关键词匹配走向能力匹配，增加证据等级
4. **最后优化 orchestrator**：加入状态判断和质量检查

---

## 核心认识

> 一个 Workflow Agent 真正难的，不是把模块串起来，而是让中间表示干净、让判断不过度、让生成不越界。

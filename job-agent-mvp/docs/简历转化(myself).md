## 3. 如何把这次优化转化为简历上的资本

### 3.1 简历里不能只写“做了一个 Agent 项目”

如果只是写“做了一个简历优化 Agent”，这个描述太泛，容易被面试官理解成普通 prompt 串联项目。
真正有价值的地方，不是“做了 4 个模块”，而是：

* 你识别了 LLM Workflow 中哪些地方最容易失真
* 你如何用工程手段让输出更可信
* 你如何把优化从“prompt 微调”提升到“结构设计”和“边界治理”

也就是说，这次优化最能转化成简历资本的，不是功能本身，而是**工程思维**。

---

### 3.2 这次优化最值得提炼成简历亮点的三个方向

#### 方向一：把普通 LLM 流程升级为证据驱动 Workflow

这是最核心的亮点。

你可以突出：

* 不是简单串联 JD 分析、匹配、改写
* 而是把系统重构为“JD 标准层 → 候选人事实层 → 证据驱动匹配 → 事实白名单改写”
* 让下游生成建立在结构化证据和边界约束上，而不是建立在关键词碰撞上

这是典型的 Agent/Workflow 工程视角，面试官通常会认可这种“不是只会调 prompt，而是会管中间表示和推理边界”的能力。

#### 方向二：设计事实边界控制机制

这是项目最有辨识度的亮点。

你可以强调：

* 发现改写模块会虚构数字、脑补功能、用 JD 术语拔高候选人能力
* 通过禁止规则、事实白名单输入、`added_facts` 自查，把 rewrite 从“追求更强表达”切换成“追求可信输出”
* 将“LLM 幻觉风险”具体落到产品级约束设计上

这个点很适合打动面试官，因为它体现了你对 LLM 落地风险的理解，而不是停留在 demo 层。

#### 方向三：把 candidate_profile 设计成中间表示层

这是偏工程实现层面的亮点。

你可以表达为：

* 原系统只是按换行切分文本，导致 skills、projects、experiences 污染严重
* 优化后将候选人材料转成结构化事实对象，独立提取技能点、项目对象、缺失项和基础信息
* 让下游模块围绕稳定的中间表示工作，而不是直接消费脏文本

这个点能体现你理解 Workflow Agent 中“中间表示质量决定上限”的工程本质。

---

### 3.3 可直接写进简历的项目表述

下面给出几版可以直接改写进简历的表述。

#### 版本 A：偏项目综述

* 设计并迭代一个面向求职场景的 Workflow Agent，完成 JD 解析、候选人事实结构化、匹配分析与安全改写全流程；通过 schema 校验、证据分级与事实边界约束，推动系统从“流程跑通”升级为“输出可信”。

#### 版本 B：偏工程优化

* 主导 Job Agent MVP 第一轮质量优化，重构候选人结构化、JD 拆解、匹配判断与改写四个模块；将系统从关键词驱动流程升级为基于结构化事实和证据等级的 Workflow Agent。

#### 版本 C：偏 LLM 约束设计

* 针对 LLM 在简历改写场景中的事实漂移问题，设计 `added_facts` 自审机制、事实白名单输入和禁止规则集，消除虚构量化指标与脑补功能，显著提升生成结果的可用性与可信度。

#### 版本 D：偏中间表示与架构

* 将候选人文本解析从逐行切分升级为结构化中间表示，独立抽取技能点、项目对象、基础信息与缺失项，解决下游 match/rewrite 受脏文本污染的问题，提升整个 pipeline 的分析稳定性。

---

### 3.4 面试时最值得展开讲的能力点

如果在面试中展开，这次优化最适合重点讲以下三类能力：

#### （1）需求抽象能力

不是把任务理解成“改 prompt”，而是先明确真实用户需求是“可信判断 + 安全改写”。

#### （2）Agent/Workflow 设计能力

理解一个 Workflow Agent 真正难的不是把模块串起来，而是：

* 中间表示是否干净
* 判断是否过度
* 生成是否越界

#### （3）LLM 风险治理能力

你不是只让模型“更会写”，而是通过 schema、证据等级、事实白名单、自审字段去限制模型输出边界。

这三类能力，比“会调用 API、会写 prompt”更容易打动面试官。

---

## 结论

第一次优化最大的收获，不是让某个模块看起来更强，而是明确了这个项目真正的产品需求：

> **用户需要的不是一个会自动写简历的 Agent，而是一个能基于真实经历完成可信判断、并在事实边界内优化表达的 Workflow Agent。**

因此，这次优化的本质也不是“优化模块表现”，而是：

* 让 `candidate_profile` 成为可信中间表示
* 让 `jd_analysis` 成为可执行的岗位标准
* 让 `match` 成为证据驱动的能力判断
* 让 `rewrite` 成为可安全使用的表达优化

而这正是这次优化最值得转化为简历资本的地方：
你不仅做了一个 Agent 项目，还开始具备了 **Workflow 设计、事实边界控制、证据驱动判断和 LLM 风险治理** 的工程意识。
